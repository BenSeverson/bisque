# Static Analysis & Build Checks Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add 7 static analysis and build-quality checks to the CI pipeline and local build: clang-tidy, compiler warnings, clang-format, Prettier, cppcheck, SPIFFS size validation, and firmware binary size tracking.

**Architecture:** All checks run as separate CI jobs (or steps within existing jobs) in `.github/workflows/build.yml`. Compiler warnings are set in `CMakeLists.txt`. Prettier is added as a dev dependency in `web_ui/`. Two new shell scripts handle SPIFFS and binary size checks.

**Tech Stack:** GitHub Actions, ESP-IDF v6.0, clang-tidy, clang-format, cppcheck, Prettier, bash

### Task 1: Add compiler warning flags to CMakeLists.txt

**Files:**
- Modify: `CMakeLists.txt`

- [ ] **Step 1: Add warning flags after project()**

Add `-Wall -Wextra -Werror` for project sources only, with `-Wno-unused-parameter` since ESP-IDF callback signatures often have unused params.

```cmake
# After project(bisque) line, add:

# Strict warnings for project sources (does not affect ESP-IDF or LVGL)
list(APPEND common_warnings -Wall -Wextra -Werror -Wno-unused-parameter)
foreach(comp IN ITEMS
    app_config cone_table display firing_engine history
    pid_control safety thermocouple web_server wifi_manager)
    idf_component_get_property(comp_lib ${comp} COMPONENT_LIB)
    target_compile_options(${comp_lib} PRIVATE ${common_warnings})
endforeach()
idf_component_get_property(main_lib main COMPONENT_LIB)
target_compile_options(${main_lib} PRIVATE ${common_warnings})
```

Note: We target each component individually via `idf_component_get_property` + `target_compile_options` instead of `add_compile_options()` to avoid applying warnings to ESP-IDF/LVGL sources.

- [ ] **Step 2: Build locally to check for warnings**

Run: `idf.py build 2>&1 | grep -E "warning:|error:" | head -40`

Expected: Either clean build, or a list of warnings to fix. If there are warnings, fix them before proceeding.

- [ ] **Step 3: Fix any warnings found**

Fix each warning. Common ones to expect:
- Unused variables: remove or cast to `(void)var;`
- Missing switch cases: add default case
- Sign comparison: use matching types

- [ ] **Step 4: Verify clean build**

Run: `idf.py build`
Expected: BUILD SUCCESSFUL with zero warnings (since `-Werror` is set)

- [ ] **Step 5: Commit**

```bash

# Plus any fixed source files
```

### Task 6: Add SPIFFS partition size check

**Files:**
- Create: `scripts/check-spiffs-size.sh`
- Modify: `.github/workflows/build.yml`

The SPIFFS `storage` partition is `0xE0000` = 917,504 bytes. The check should fail if compressed web assets exceed 90% of this (825,753 bytes), leaving headroom.

- [ ] **Step 1: Create the check script**

Write to `scripts/check-spiffs-size.sh`:

```bash
#!/usr/bin/env bash
set -euo pipefail

SPIFFS_DIR="${1:-spiffs_data/www}"
# From partitions.csv: storage partition is 0xE0000 = 917504 bytes
PARTITION_SIZE=917504
# Warn at 80%, fail at 95%
WARN_THRESHOLD=$(( PARTITION_SIZE * 80 / 100 ))
FAIL_THRESHOLD=$(( PARTITION_SIZE * 95 / 100 ))

ACTUAL_SIZE=$(du -sb "$SPIFFS_DIR" | cut -f1)
PERCENT=$(( ACTUAL_SIZE * 100 / PARTITION_SIZE ))

echo "SPIFFS usage: ${ACTUAL_SIZE} / ${PARTITION_SIZE} bytes (${PERCENT}%)"

if [ "$ACTUAL_SIZE" -gt "$FAIL_THRESHOLD" ]; then
    echo "::error::SPIFFS partition usage at ${PERCENT}% — exceeds 95% limit"
    exit 1
elif [ "$ACTUAL_SIZE" -gt "$WARN_THRESHOLD" ]; then
    echo "::warning::SPIFFS partition usage at ${PERCENT}% — approaching limit"
fi

echo "SPIFFS size check passed"
```

- [ ] **Step 2: Make it executable**

Run: `chmod +x scripts/check-spiffs-size.sh`

- [ ] **Step 3: Add to CI after web asset gzip step**

Add in the `build` job after the "Gzip web assets" step:

```yaml
      - name: Check SPIFFS size
        run: ./scripts/check-spiffs-size.sh spiffs_data/www
```

- [ ] **Step 4: Test locally**

### Task 7: Add firmware binary size tracking

**Files:**
- Create: `scripts/check-firmware-size.sh`
- Modify: `.github/workflows/build.yml`

The OTA partitions are `0x180000` = 1,572,864 bytes each. The firmware binary must fit in one OTA slot.

- [ ] **Step 1: Create the check script**

Write to `scripts/check-firmware-size.sh`:

```bash
#!/usr/bin/env bash
set -euo pipefail

BINARY="${1:-build/bisque.bin}"

if [ ! -f "$BINARY" ]; then
    echo "::error::Firmware binary not found: $BINARY"
    exit 1
fi

# From partitions.csv: OTA partitions are 0x180000 = 1572864 bytes
PARTITION_SIZE=1572864
WARN_THRESHOLD=$(( PARTITION_SIZE * 80 / 100 ))
FAIL_THRESHOLD=$(( PARTITION_SIZE * 95 / 100 ))

ACTUAL_SIZE=$(stat -f%z "$BINARY" 2>/dev/null || stat -c%s "$BINARY")
PERCENT=$(( ACTUAL_SIZE * 100 / PARTITION_SIZE ))

echo "Firmware size: ${ACTUAL_SIZE} / ${PARTITION_SIZE} bytes (${PERCENT}%)"

if [ "$ACTUAL_SIZE" -gt "$FAIL_THRESHOLD" ]; then
    echo "::error::Firmware binary at ${PERCENT}% of OTA partition — exceeds 95% limit"
    exit 1
elif [ "$ACTUAL_SIZE" -gt "$WARN_THRESHOLD" ]; then
    echo "::warning::Firmware binary at ${PERCENT}% of OTA partition — approaching limit"
fi

echo "Firmware size check passed"
```

- [ ] **Step 2: Make it executable**

Run: `chmod +x scripts/check-firmware-size.sh`

- [ ] **Step 3: Add to CI after firmware build**

Add in the `build` job after the firmware build step:

```yaml

```

- [ ] **Step 4: Test locally**

After all tasks, the workflow should look like:

```
lint-web  ──┐
             ├──→ build ──→ clang-tidy ──→ cppcheck ──→ size checks ──→ upload artifact
lint-c   ──┘
```

Jobs:
- `lint-web`: typecheck → eslint → prettier check
- `lint-c`: clang-format check
- `build` (needs lint-web + lint-c): build web → gzip → SPIFFS size check → build firmware → firmware size check → clang-tidy → cppcheck → upload artifact
